ed-wings-management-system documentation

Dokumentacja techniczna aplikacji "Edwings Company Management"

Podstawowe Komendy

  • Migracja bazy danych: npx prisma migrate dev --name AddLogger
  • Uruchamianie projektu: npm run start:dev
  • Interfejs graficzny do zarządzania bazą danych: npx prisma studio -p postgresql://{ConnectionString}/postgres
  • Tworzenie dokumentacji: npm compodoc
  • Uruchomienie testów: npm test
  • Dokumentacja API: Dostępna pod adresem http://localhost:3000/api po uruchomieniu aplikacji lokalnie
  • Instalacja prisma-dbml-generator: npm i prisma-dbml-generator - Przekształca schemat Prismy na schemat .dbml
  • Inicjalizacja wartości w bazie danych: npx prisma db seed

Struktura aplikacji

Aplikacja została podzielona na dwa główne foldery:

  • Prisma:
    • dbml - Schemat bazy danych w formacie .dbml
    • migrations - Zawiera wszystkie migracje bazy danych
    • schema.prisma - Główny schemat bazy danych
    • seed.ts - Skrypt do inicjalizacji danych testowych w bazie danych
  • Src:
    • Auth:
      • decorators - Custom dekoratory, np. role, porównywanie wartości w DTO, pobieranie danych użytkownika z requesta
    • Common:
      • helpers - Uniwersalne funkcje, np. hashowanie haseł, porównywanie klas
      • validation - Filtry do obsługi wyjątków
    • Models:
      • dto - Folder zawierający DTO dla danego modelu
      • .controller.ts - Obsługa żądań HTTP i zwracanie odpowiedzi klientowi
      • .module.ts - Grupowanie powiązanych elementów takich jak kontrolery, serwisy i inne moduły
      • .service.ts - Implementacja logiki biznesowej, przetwarzanie danych, interakcje z bazami danych
