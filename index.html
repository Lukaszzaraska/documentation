ed-wings-management-system documentation

Dokumentacja techniczna aplikacji "Edwings company management"

Podstawowe komendy

Migracja bazy danych- npx prisma migrate dev --name AddLogger

Uruchamia projekt - npm run start:dev

Włącza interfejs graficzny do zarządzania bazą danych - npx prisma studio -p postgresql://{Connectionstring}/postgres

Tworzenie dokumnetacji - compodoc -p tsconfig.doc.json

Uruchomienie testów - npm test

Dokumentacja Api dostępna pod adresem http://localhost:3000/api po uruchomieniu aplikacji lokalnie

npm i prisma-dbml-generator

Inicjalizuje wartości w bazie danych - npx prisma db seed

Aplikacja została podzielona na dwa główne foldery, poniżej zostały opisane najważniejsze pliki i ich założenia.

  • prisma
    1. dbml - schemat bazy danych w formacie .dbml
    2. migrations - zawiera wszystkie migracje bazy danych
    3. schema.prisma - zawiera główny schemat na podstawie którego jest generowana baza danych
    4. seed.ts - Stosowany do testowania, pozwala zainicjalizować dane testowe w bazie danych
  • src
    1. Auth
      1. decorators - zawiera custom dekoratory, takie jak role, porównywanie wartości w dto, pobieranie danych o użytkowniku z requesta
    2. common
      1. helpers - zawiera uniwersalne funkcje. Np. hashowanie haseł, porównywanie klas
      2. validation - zawiera wszystkie filtry służące do obsługi wyjątków
    3. models - zawiera wszystkie modele używane w aplikacji. Każdy model posiada ten sam schemat
      1. dto - folder zawierajacy wszystkie pliki dto używane w danym modelu
      2. .controller.ts - odpowiedzialne są za obsługę przychodzących żądań HTTP i zwracanie odpowiedzi do klienta.
      3. .module.ts - są kontenerami, które grupują powiązane elementy takie jak kontrolery, serwisy, a także inne moduły
      4. .service.ts - są używane do implementacji logiki biznesowej. To tutaj odbywa się przetwarzanie danych, interakcje z bazami danych i wykonywanie operacji biznesowych.
